# Constitution — cwebp 项目原则

> 不可变约束。任何 spec / plan / 实现都不得违反本文件。若需破例，先修改本文件并记录原因。

## 核心原则

### P1. 隐私优先，本地处理

所有图像处理**必须**在用户设备上完成，不得将用户图片上传到任何服务器。MVP 不引入任何后端。这既是用户信任的基础，也是我们和同类在线转换站的差异化卖点。

**检验方式**：打开 DevTools → Network 面板，完成一次完整转换流程，不应看到任何非静态资源请求。

### P2. 零安装门槛

MVP 必须以"访问 URL 即用"的形态交付，不要求用户安装任何客户端、扩展或运行时。后续桌面版独立演进，不破坏 Web 版。

### P3. 聚焦 WebP

工具目标 = **输入 PNG / JPEG / JPG，输出 WebP**。不在 MVP 阶段加入 AVIF、HEIC、GIF 动图、PDF 等其他格式。功能蔓延是 MVP 最大的敌人。

### P4. 可视化界面是必需品，不是附属

本项目不是一个"带界面的 CLI"，UI 的质量和 CLI 的质量同等重要。拖放、进度、预览、体积对比必须具备。

### P5. 可演进，不过度设计

MVP 代码结构要让后续升级为 Tauri 桌面版、加入 Web Worker、接入 WASM 编码器时尽量少改动。但不提前为假想需求抽象。"三段相似代码胜过一个过早的抽象。"

## 技术红线

1. **不用 Electron**。2026 年已被 Tauri 全面超越（包体 25x、启动 4x、内存 -70%）。
2. **不引入 Node.js 后端**，MVP 阶段静态文件即可。
3. **不私建 WebP 编码器**，一律用 libwebp（通过浏览器 Canvas API 或 WASM 封装）。
4. **不追踪用户**，不接统计 SDK，不发远程日志。

## 质量底线

- 所有 UI 文案优先中文（用户母语）。
- TypeScript `strict: true`，不允许 `any` 偷懒（除第三方类型缺失外）。
- 关键路径（队列、转换、下载）必须可见进度，不能静默失败。
- 任何一张图转换失败不得阻塞其他图。

## 范围外（明确排除）

以下功能明确**不在**本项目范围内，用户反复提出也不在 MVP 做：

- 图片裁剪、旋转、滤镜、水印
- 账号系统 / 云端同步 / 分享链接
- 付费墙 / 订阅
- 多语言（Phase 2+ 再考虑）
- 移动端 App（Web 自适应即可）

## 决策变更流程

任何对本文件的修改都要：

1. 在 PR / commit 里注明被修改的条款编号。
2. 说明变更原因（用户反馈、技术约束、市场变化等）。
3. 同步更新 [../../CLAUDE.md](../../CLAUDE.md) 和受影响的 spec.md / plan.md。
