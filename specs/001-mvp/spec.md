# Spec — cwebp MVP

> **WHAT & WHY**，不写 HOW。HOW 在 [plan.md](plan.md)。

## 1. 问题陈述

用户手里有一批 PNG / JPEG 图片（常见于设计稿切图、截图、相机导出），想批量转成 WebP 以：

- 降低网页加载体积（WebP 相比 PNG 平均 -26%，相比 JPEG -25%～-34%）。
- 统一素材格式。

当前可选项的痛点：

- **在线转换站**（TinyPNG / CloudConvert 等）：要上传，隐私有顾虑；免费有额度；批量操作繁琐。
- **cwebp CLI**：不会写命令行的用户用不了；没有可视化反馈。
- **Squoosh**：单张处理体验极好，但**没有批量**。
- **Sharp / libvips**：偏后端开发者。

**我们的定位**：面向普通用户和前端开发者的**"Squoosh + 批量"**，强调隐私与零门槛。

## 2. 目标用户

| 画像 | 场景 | 期望 |
|---|---|---|
| 前端开发者 | 把设计稿导出的 PNG 切图批量转为 WebP 上线 | 快速、批量、质量可调、体积反馈 |
| 独立博客作者 | 把相机 JPEG 放到文章前先瘦身 | 拖进去就出结果，无需学习 |
| 设计师 | 给客户交付前统一格式 | 预览对比、无损选项 |

## 3. 用户故事（MVP 范围）

### US-1 单张转换（最小路径）

> 作为用户，我把一张 PNG 拖到页面上，看到它被转为 WebP，点击即可下载。

**验收**：
- 拖入或点击上传一张图，2 秒内出现转换结果卡片。
- 卡片显示：缩略图、原文件名、原格式、原大小、输出大小、节省百分比、下载按钮。
- 点击下载按钮得到 `{原文件名}.webp`。

### US-2 批量转换

> 作为用户，我一次拖入 20 张图（混合 PNG 和 JPEG），全部自动转换，我可以逐个下载，也可以一键"下载全部（ZIP）"。

**验收**：
- 支持一次拖入多文件或文件夹。
- 每张图独立展示进度与结果。
- "全部下载（ZIP）" 按钮打包所有已成功转换的 WebP。

### US-3 质量调整

> 作为用户，我拖动质量滑块看到预览缩略图和输出大小实时变化，决定一个满意的值后应用到所有文件。

**验收**：
- 质量滑块范围 0–100，默认 80。
- 滑块停止拖动 300ms 后，所有文件自动重新编码并刷新体积。
- 质量值持久化到 `localStorage`，下次打开保留。

### US-4 无损模式

> 作为用户，对于带透明通道的 PNG logo，我勾选"无损"，输出 WebP 保留透明且视觉无差。

**验收**：
- 存在一个"无损模式"开关。
- 开启时，质量滑块禁用（或提示无效），编码走 WebP lossless。
- 透明 PNG 转换后透明通道完整。

### US-5 清晰的失败反馈

> 作为用户，如果某张图失败（损坏、格式不支持），我能看到明确错误，不影响其他图继续转换。

**验收**：
- 失败的文件卡片显示红色状态与错误原因（如"文件已损坏"、"不支持的格式"）。
- 队列中其他文件不受影响继续转换。

## 4. 范围外（明确不做）

- 图像编辑（裁剪、滤镜、旋转）—— 交给专门工具。
- AVIF / HEIC 输出 —— 只做 WebP。
- 其他输入格式（GIF、SVG、PSD、RAW）—— 只认 PNG / JPEG / JPG。
- 账号、历史记录、云端存储。
- 单张图 > 50 MB 或总量 > 500 MB 的极端场景（性能退化容忍度低，明确提示）。

## 5. 非功能需求

| 维度 | 要求 |
|---|---|
| 性能 | 1920×1080 JPEG ≤ 1s / 张；4K PNG ≤ 3s / 张（中端笔记本） |
| 隐私 | 任何用户图像字节不离开用户设备（Constitution P1） |
| 浏览器兼容 | Chrome/Edge/Safari/Firefox 最新两个大版本 |
| 可用性 | 首屏 LCP < 2s；操作反馈 < 100ms |
| 可访问性 | 关键操作可键盘完成；色彩对比度符合 WCAG AA |
| 错误处理 | 任一文件失败不得阻塞其他文件 |

## 7. 开放问题（推进前需要决策）

1. **质量默认值 80 vs 85**：业内文章多推 85，但 80 在肉眼几乎无差的同时体积更小。倾向 **80**，等 Phase 4 实测图集后再定。
2. **ZIP 打包是否 MVP 必需**：引入 `jszip` 约 +40 KB。倾向 **必需**，否则 10+ 张图下载体验崩溃。
3. **File System Access API 是否优先使用**：能直接写回源目录体验好，但 Safari/Firefox 不支持。MVP **先不启用**，用标准下载流。

这三条等进入 plan.md 时做最终裁决。
